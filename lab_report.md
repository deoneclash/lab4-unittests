# lab4-unittests

# Отчёт по лабораторной работе №4 — Написание Unit-тестов

## 1. Цели и задачи тестирования
- **Цель:** изучить методику написания и запуска unit-тестов для простого Python-модуля.
- **Задачи:**
  - Реализовать функцию вычисления площади прямоугольника.
  - Написать покрывающие unit-тесты, отражающие обычные, пограничные и ошибочные ситуации.
  - Продемонстрировать процесс запуска тестов и получение отчёта.

---

## 2. Описание тестируемого продукта
Модуль `rectangle.py` содержит функцию:

- `area(a, b)` — возвращает площадь прямоугольника (float).

**Требования к функции:**
- Параметры должны быть числами (`int` или `float`).
- Значения параметров должны быть **≥ 0**.
- При некорректных типах должно вызываться исключение `TypeError`.
- При отрицательных значениях — исключение `ValueError`.

---

## 3. Область тестирования
Покрываются следующие категории тестов:

- Корректные положительные значения (целые и дробные).
- Нулевые значения.
- Очень большие числа.
- Негативные значения (ожидается `ValueError`).
- Неподходящие типы данных (ожидается `TypeError`).

---

## 4. Стратегия тестирования
- Тип тестирования: **модульное функциональное тестирование (unit testing)**.
- Инструмент: встроенный модуль Python **`unittest`**.
- Для каждого типа проверяемого поведения создаётся отдельный тест-метод.
- Критерий успешности: **все тесты проходят без ошибок и падений**.

---

## 5. Критерии приёмки
- Все тесты в `test_rectangle.py` должны выполняться успешно (0 ошибок, 0 неудач).
- Функция должна корректно обрабатывать корректные данные.
- При неверных типах или отрицательных значениях должны выбрасываться соответствующие исключения.

---

## 6. Ожидаемые результаты
- Подробный отчёт о выполнении unit-тестов (данный документ).
- Логи тестовой системы — сообщение вида `OK` после выполнения тестов.
- Подтверждение корректности работы функции `area()`.

---

## 7. Фактический результат выполнения тестов
Все тесты были успешно выполнены.  
Функция `area()` корректно вычисляет площадь для стандартных, нулевых и больших значений.  
Для отрицательных параметров и неверных типов данных тесты подтверждают корректное возбуждение исключений.  
Итоговый статус тестов: **OK**.

---

## 8. Вывод
В ходе выполнения лабораторной работы мы изучили основы модульного тестирования, создали unit-тесты для функции расчёта площади прямоугольника и проверили корректность её работы.  
Тестирование подтвердило правильность реализации и продемонстрировало практическое применение `unittest` для автоматизации проверки кода.
